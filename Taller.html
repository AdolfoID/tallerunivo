🤖 ¿Qué es Programar?

Programar es como darle una receta detallada a una computadora

💡 Instrucciones Claras

Las computadoras son muy literales. Si le dices a un robot "trae agua", no sabrá de dónde, cuánta, ni en qué recipiente.

Ejemplo: En vez de "trae agua", debes decir: "camina 5 pasos, gira 90° a la derecha, toma el vaso, camina 3 pasos..."

🎯 Orden Preciso

El orden importa. No puedes freír un huevo antes de romperlo. Cada paso debe estar en secuencia.

Ejemplo: Para hacer café: 1. Calentar agua, 2. Poner café, 3. Verter agua, 4. Revolver.

🧩 Resolver Problemas

Programar no es memorizar código, es aprender a dividir problemas grandes en pasos pequeños.

Ejemplo: "Organizar fiesta" → Invitados + Comida + Decoración + Música

🧠 Pensamiento Computacional

Los 4 pilares para resolver cualquier problema

1️⃣ Descomposición

Dividir un problema gigante en pedazos manejables

Ejemplo práctico:
🎯 Problema: Organizar una fiesta
📝 Partes:
• Lista de invitados
• Comida y bebidas
• Decoración
• Música y entretenimiento

2️⃣ Reconocimiento de Patrones

Identificar qué se repite para no reinventar la rueda

Ejemplo práctico:
📧 Problema: Enviar 100 emails
🔄 Patrón:
1. Mismo formato para todos
2. Solo cambia el nombre
3. Usar plantilla

3️⃣ Abstracción

Enfocarse en lo importante, ignorar los detalles innecesarios

Ejemplo práctico:
🗺️ Mapa del metro:
✅ Incluye: Estaciones, líneas, conexiones
❌ Omite: Calles, edificios, árboles
(Solo lo esencial para viajar)

🎯 Desafío: El Problema del Río

Un granjero necesita cruzar un río con un lobo, una cabra y una col. El bote solo puede llevar al granjero y una cosa más.

Reglas:

  • Si deja solos al lobo y la cabra → 🐺 se come 🐐
  • Si deja solos a la cabra y la col → 🐐 se come 🥬
A) Llevar lobo, regresar, llevar cabra, regresar, llevar col
B) Llevar cabra, regresar, llevar lobo, traer cabra, llevar col, regresar, llevar cabra
C) Llevar col, regresar, llevar cabra, regresar, llevar lobo

🎉 ¡Correcto! Análisis de la solución:

Paso a paso:

  1. Llevar cabra al otro lado
  2. Regresar solo
  3. Llevar lobo al otro lado
  4. Traer cabra de regreso
  5. Llevar col al otro lado
  6. Regresar solo
  7. Llevar cabra al otro lado

✅ ¿Por qué funciona? Siempre evita que queden juntos los que no deben.

🔁 Repeticiones (Bucles)

Cuando necesitas hacer algo muchas veces

🔄 Repetir N veces

Ejemplo: Aplaudir 10 veces

REPETIR 10 veces:
  👏 Aplaudir
FIN REPETIR
¿Cuándo usarlo? Cuando sabes exactamente cuántas veces quieres repetir.

⏳ Mientras (condición)

Ejemplo: Llenar un vaso con agua

MIENTRAS vaso no esté lleno:
  💧 Agregar agua
FIN MIENTRAS
¿Cuándo usarlo? Cuando no sabes cuántas veces, pero tienes una condición de parada.

📋 Para cada elemento

Ejemplo: Saludar a cada persona

PARA CADA persona en lista:
  👋 Decir "Hola"
FIN PARA CADA
¿Cuándo usarlo? Cuando quieres procesar todos los elementos de una lista.

🎯 Pregunta: ¿Cuántas veces?

Si necesitas imprimir tu nombre 1000 veces, ¿cuál sería la mejor opción?

A) Escribir "imprimir mi nombre" 1000 veces en el código
B) Usar un bucle: REPETIR 1000 veces { imprimir nombre }
C) Copiar y pegar 1000 veces en un documento

✅ ¡Exacto! Los bucles son más eficientes:

Ventajas de usar bucles:

  • 📏 Menos código: Solo unas líneas
  • ⚡ Más rápido: La computadora repite rápidamente
  • 🔧 Fácil cambiar: Si quieres 2000 veces, solo cambias un número
  • 🐞 Menos errores: Solo tienes que escribir correctamente una vez

📦 Variables: Cajas de Memoria

Las variables son como cajas donde guardamos información

🏷️ ¿Qué es una Variable?

Es un espacio en la memoria con un nombre, donde guardamos un dato que puede cambiar.

Ejemplo en código:
edad = 15
nombre = "María"
estaLloviendo = true
💡 Tip: Los nombres de variables deben ser descriptivos (ej: "edad", no "x")

🔢 Tipos de Datos

Números 42, 3.14, -7
Texto "Hola", "Programación"
Booleanos verdadero o falso
Listas [1, 2, 3, 4, 5]

➕ Operaciones con Variables

Podemos hacer cálculos y modificar los datos:

// Sumar 1 a la edad
edad = edad + 1

// Duplicar el puntaje
puntaje = puntaje * 2

// Concatenar texto
nombreCompleto = "Dr. " + nombre

🎯 Desafío: Buscar un Número

Tienes una lista de números del 1 al 100 ordenados. ¿Cuál es la forma más rápida de encontrar el número 73?

A) Revisar uno por uno desde el 1
B) Ir al medio (50), si 73 es mayor, buscar en la mitad derecha. Repetir.
C) Empezar desde el 100 hacia atrás

🎉 ¡Correcto! Búsqueda Binaria

¿Por qué es más eficiente?

Búsqueda lineal:
• Revisa 73 elementos
• Más lento
• Más pasos
Búsqueda binaria:
• Revisa ~7 elementos
• Más rápido
• Menos pasos

💡 Analogía: Es como buscar una palabra en el diccionario: no empiezas desde la A, abres por la mitad.

🐛 Depuración: Cazar Bugs

Los errores son normales. Lo importante es aprender a encontrarlos.

🐞 ¿Qué es un Bug?

Un error en el programa que hace que no funcione como esperábamos.

Dato curioso: El término "bug" viene de 1947, cuando una polilla real causó un error en la computadora Harvard Mark II.

🚨 Tipos de Errores

Sintaxis Escribiste mal el código
Lógica El código corre, pero hace lo incorrecto
Ejecución El programa se detiene inesperadamente

🔧 Cómo Depurar

1️⃣ Leer el mensaje de error cuidadosamente
2️⃣ Revisar el código paso a paso
3️⃣ Usar "print" para ver valores intermedios
4️⃣ Simplificar el problema y probar partes pequeñas

🎮 Encuentra el Bug

Este programa debe sumar dos números, pero tiene un error. ¿Puedes encontrarlo?

numero1 = 5
numero2 = 3
resultado = numero1 - numero2
mostrar("El resultado es: " + resultado)
